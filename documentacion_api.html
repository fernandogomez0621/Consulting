modulos.datos.carga_s3

Funciones de carga de datos desde Amazon S3. Todos los CSVs, JSONs y modelos pickle se leen desde el bucket configurado. Las funciones usan @st.cache_data y @st.cache_resource para evitar lecturas repetidas.
def leer_csv(key, **kwargs)

Lee un archivo CSV desde S3 y lo devuelve como DataFrame. Decodifica con UTF-8-SIG para manejar BOM.

@cached (1h)
ParametroTipoDescripcion
keystrRuta del archivo dentro del bucket (ej: datos_procesados/X_limpio.csv)
**kwargsdictArgumentos adicionales para pd.read_csv() (ej: parse_dates=['Date'])

Retorna pd.DataFrame

def leer_json(key)

Lee un archivo JSON desde S3 y lo devuelve como diccionario. Si falla, retorna un dict con la clave "error".

@cached (1h)
ParametroTipoDescripcion
keystrRuta del JSON en S3

Retorna dict

def leer_pickle(key)

Lee un archivo pickle desde S3 y deserializa el objeto.

@cached resource (1h)
ParametroTipoDescripcion
keystrRuta del pickle en S3 (ej: modelos/prophet_equipo1.pkl)

Retorna object (modelo deserializado)

agente.DatosS3

class Cache de JSONs desde S3 para el agente. Cada propiedad carga y cachea un JSON especifico. No usa st.cache porque opera fuera de Streamlit.

Propiedades disponibles (cada una retorna dict):

estadisticas, normalidad, estacionariedad, correlaciones,
causalidad_granger, anomalias, volatilidad, vif,
mutual_information, evaluacion_features, feature_importance,
segmentacion_covid, rezagos, resumen_eda, comparacion_modelos,
info_modelos, residuos, validacion_dist, prophet_resultados,
resumen_modelamiento, proyecciones

agente.ejecutar_tool

def ejecutar_tool(nombre, parametros, datos)

Router principal de tools del agente. Recibe el nombre de la tool, sus parametros, y el objeto DatosS3. Lee los JSONs correspondientes y devuelve el resultado como string JSON.

ParametroTipoDescripcion
nombrestrNombre de la tool (ej: 'consultar_eda', 'proyectar_costos')
parametrosdictParametros enviados por el LLM
datosDatosS3Instancia del cache de datos

Retorna str (JSON serializado)

Tools soportadas: consultar_eda, consultar_correlaciones, consultar_features, consultar_modelo, consultar_prophet, proyectar_costos, consultar_segmentacion, consultar_volatilidad, comparar_modelos

agente.Agente

class Agente conversacional con Claude via Amazon Bedrock. Maneja el loop de tool use automaticamente: envia mensaje -> si Claude pide tool -> la ejecuta -> devuelve resultado -> repite hasta respuesta final.
def __init__(self)

Inicializa el cliente Bedrock, el cache de datos S3, y el historial de conversacion vacio.

def consultar(self, mensaje_usuario)

Envia un mensaje al agente y obtiene la respuesta final. Maneja multiples rondas de tool use si es necesario. El historial se acumula para mantener memoria conversacional.

ParametroTipoDescripcion
mensaje_usuariostrPregunta del usuario

Retorna str (respuesta en texto plano)

def reiniciar(self)

Limpia el historial de conversacion. Util para empezar una nueva sesion.
